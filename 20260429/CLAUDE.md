# CLAUDE.md - プロジェクト開発規約

## プロジェクト概要

**目的**: Claude Code を用いた高効率開発環境の構築と、チーム間でのベストプラクティス統一化  
**スタック**: Node.js/TypeScript（推奨）、Python、Go 等、言語依存なし  
**チームサイズ**: 1-5名（単一エージェント + オプション subagent）  

---

## コーディング規約

### 命名規則

- **ファイル**: kebab-case （例: `user-service.ts`, `data-processor.py`）
- **ディレクトリ**: kebab-case （例: `/src/api-handlers/`)
- **変数・定数**: camelCase （例: `userCount`, `MAX_RETRIES`）
- **クラス**: PascalCase （例: `UserService`, `DataProcessor`）
- **関数**: camelCase （例: `fetchUserData()`, `validateEmail()`)
- **プライベートメンバ**: アンダースコアプレフィックス （例: `_internalCache`)

### フォーマット

- **インデント**: 2 スペース（Prettier デフォルト）
- **行幅**: 100 文字を超えない
- **クォート**: ダブルクォート推奨（`"` ）
- **セミコロン**: 末尾に常に付与
- **末尾のカンマ**: ES5 互換性のため、オブジェクト・配列で常に付与

### コメント

- **コメントは「何をするか」ではなく「なぜするか」を記述**
- 明らかなコード（`count++` など）にはコメント不要
- 隠れた制約、バグ回避、パフォーマンス工夫の場合のみ 1 行コメント

例：
```typescript
// 並行リクエスト数を制限しないと OOM になる（issue #1234）
const CONCURRENT_LIMIT = 10;
```

### エラーハンドリング

- **システム境界のみで入力検証** （外部 API、ユーザー入力）
- **内部関数は入力値を信頼** （事前条件を呼び出し側で保証）
- **フォールバック・リトライは具体的な失敗ケースのみ** （ネットワークタイムアウト等）

## 禁止事項・動作制限

### 絶対禁止

- [ ] 🚫 本番環境 `.env` ファイルの編集・コミット
- [ ] 🚫 `git reset --hard`, `git push --force` （特に共有ブランチ）
- [ ] 🚫 `/dev/null` ディレクトリ内への書込（サンドボックス外）
- [ ] 🚫 `npm audit fix --force` （依存関係の破損リスク）
- [ ] 🚫 **セキュリティ関連の MCP サーバー設定を独断で追加**（チーム審査必須）

### 警告・確認必須

⚠️ 以下の操作は実行前に **必ずユーザーに確認** を取ってください：
- 本番環境への deploy
- 50 行以上の削除・リファクタリング
- データベーススキーマ変更
- 外部 API キー・認証情報の変更
- MCP サーバー追加（SECURITY.md 参照）

### 制限事項

- **大規模ファイル（>10MB）の同時読込**: 最大 3 ファイルまで
- **依存関係の自動アップデート**: 禁止（package-lock.json 確認後の手動更新のみ）
- **新規ライブラリ導入**: SECURITY.md の審査プロセス経由のみ

## ファイル編集ポリシー

### 優先ルール

1. **既存ファイル編集を優先** （新規ファイル作成より）
2. **差分編集（Edit ツール）を優先** （全文上書き Write より）
3. **1 コミット = 1 つのタスク** （複数ロジックを混ぜない）

### 対象外ファイル（編集禁止）

以下のファイルは自動生成・外部管理されるため、編集しないでください：
- `package-lock.json`, `yarn.lock`, `Pipfile.lock`
- `.git/` 配下すべて
- ビルド出力: `dist/`, `build/`, `.next/`
- 環境固有: `.vscode/`, `.DS_Store`

### オプション: .gitignore に追加推奨
```
.claude/
node_modules/
.env.local
*.log
```

## テスト・ビルド・デプロイポリシー

### テスト実行

```bash
# 単体テスト（変更ファイルに関連するテストのみ）
npm test -- --testPathPattern="module-name"

# 統合テスト（マージ前に必須）
npm test

# カバレッジ確認（60% 以上を目安）
npm test -- --coverage
```

**制約**:
- テスト失敗時のコミット禁止
- 新規機能は対応テストケース必須

### ビルド・型チェック

```bash
# TypeScript 型チェック
npx tsc --noEmit

# ビルド実行（デプロイ前）
npm run build
```

**制約**:
- 型チェックエラーのマージ禁止
- ビルド成功を deploy 前提条件とする

### デプロイ

```bash
# ステージング環境への反映（事前確認）
npm run deploy:staging

# 本番環境への反映（事後報告）
npm run deploy:production
```

**制約**:
- 本番デプロイはユーザー指示必須
- 必ずステージング環境で事前テスト
- ロールバック手順を事前に確認

## コンテキスト管理ルール

### トークン監視

**Claude Code 実行時の確認**:
```
> /context
```

- **60-70%** 到達時：`/compact` 実行を検討
- **75%** 超過時：新規セッション `npm start / python main.py` で `/clear` 推奨

### ファイル読込制限

- 1 回の操作で読込むファイル数：最大 5 個
- 1 ファイルサイズ：最大 5000 行
- 大規模ログ：最初の 100 行 + 最後の 100 行のみ参照

### 参考ファイル管理

| 種別 | 保持ファイル | 説明 |
|---|---|---|
| **常時参照** | CLAUDE.md, DESIGN.md | リポジトリ固有ルール |
| **必要に応じて** | CONTEXT.md, SECURITY.md | 特定タスク用 |
| **参照禁止** | node_modules/, dist/, .git/ | コンテキスト無駄遣い |

## チーム協働ルール

### Pull Request プロセス

1. **ブランチ作成**: `feature/task-name` または `fix/issue-name`
2. **コミット**: 3 行以上のメッセージ（what/why/how）
3. **PR 前**: ローカルでテスト・ビルド実行
4. **レビュー**: 最低 1 名の承認後 merge

### 同時実行制限

- 同一ブランチで複数 Claude Code セッション実行禁止
- subagent を並行実行する場合は worktree isolation 必須（HOOKS.md 参照）

### ドキュメント

- **README.md**: プロジェクト全体の目的・セットアップ手順
- **ARCHITECTURE.md**: 主要モジュールの設計（別ファイル。CLAUDE.md に embed しない）
- **API.md**: 外部 API 仕様（OpenAPI yaml 推奨）

## トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| Claude が古い情報で意思決定 | コンテキストウィンドウ満杯 | `/compact` または `/clear` 実行 |
| テスト失敗が解決しない | キャッシュが古い | `npm test -- --clearCache` |
| 権限エラーで操作失敗 | MCP サーバー設定ミス | SECURITY.md の「許可リスト」確認 |
| Hooks が実行されない | ~/.claude/settings.json 構文エラー | `cat ~/.claude/settings.json` で妥当性検証 |
